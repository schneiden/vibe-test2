# 客戶新增與列表功能規格

## Goal
使用者可以新增客戶基本資料（姓名、電話、email），系統會保存並顯示客戶列表。

## Non-Goals
- 不包含客戶資料的編輯或刪除功能
- 不包含使用者登入或權限驗證
- 不包含資料搜尋或排序功能
- 不包含資料驗證或錯誤處理

## Schema
客戶資料結構：
```
Customer {
  name: string    // 客戶姓名
  phone: string   // 客戶電話
  email: string   // 客戶 Email
}
```

## User Flow
1. 使用者在表單中輸入客戶姓名、電話、email
2. 使用者點擊「新增客戶」按鈕
3. 系統保存客戶資料
4. 系統在頁面上顯示更新後的客戶列表

## Acceptance Criteria
- [ ] 頁面包含姓名、電話、email 三個輸入欄位
- [ ] 頁面包含「新增客戶」提交按鈕
- [ ] 點擊提交按鈕後，客戶資料會被保存
- [ ] 保存成功後，客戶資料會顯示在列表中
- [ ] 表單提交後會清空輸入欄位
- [ ] 客戶列表會顯示所有已保存的客戶資料
